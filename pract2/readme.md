# Практична робота 3: Оптимізація та аналіз енергетичних даних

Цей репозиторій містить комплексний пайплайн для обробки, оптимізації та аналізу великих наборів даних енергоспоживання. Виконано 25 спеціалізованих завдань з використанням сучасного Python-стеку, включаючи низькорівневу оптимізацію на C (Cython).

##  Технологічний стек
* **Оптимізація обчислень:** Numba, Cython, Multiprocessing.
* **Обробка даних (Pandas & Big Data):** Pandas, Polars, Dask, PyArrow (Parquet).
* **Аналіз часових рядів:** Statsmodels, Scipy.
* **Машинне навчання та Feature Engineering:** Scikit-learn, Featuretools.
* **Генерація даних:** Faker, Markovify.

##  Опис реалізованих блоків

### Блок 1: Оптимізація та паралельні обчислення
* **Numba та Cython:** Реалізовано швидкі математичні розрахунки енергетичних потреб. Numba JIT-компіляція та типізований Cython-модуль продемонстрували значне прискорення (у десятки разів) порівняно з чистим Python.
* **Multiprocessing:** Задіяно паралельні пули процесів для швидкої оцінки ефективності енерговикористання розбиттям масивів на чанки.
* **Монте-Карло:** Метод $I \approx \frac{V}{N} \sum f(x)$ реалізовано через `numba.jit` для надшвидкого прогнозування навантаження на мережу.
* **Алгоритми сортування:** Порівняно базовий алгоритм сортування, написаний через Numba, із вбудованим `numpy.sort()`.

### Блок 2: Оптимізація роботи з Pandas
* **Управління пам'яттю:** Використання параметра `chunksize` при читанні об'ємних CSV дозволило уникнути переповнення RAM.
* **Продуктивність:** Застосовано методи `.query()` та `.eval()` для оптимізації фільтрації.
* **Типізація:** Зменшено споживання пам'яті датафреймом більш ніж удвічі завдяки використанню `.astype()` (перехід від `float64` до `float32` та `category`).

### Блок 3: Часові ряди (Time Series)
* **Заповнення пропусків:** Реалізовано лінійну інтерполяцію для автоматичного відновлення втрачених даних з датчиків.
* **Агрегація та Децимація:** Застосовано метод `resample()` для тижневого усереднення та децимацію для зниження розмірності ряду.
* **Декомпозиція:** За допомогою `seasonal_decompose` часовий ряд розбито на тренд, сезонність та залишкову компоненти.

### Блок 4: Генерація та статистичний аналіз
* **Синтетичні дані:** Залучено бібліотеку `Faker` для створення профілів споживачів.
* **NLP & Тексти:** `Markovify` використано для процедурної генерації текстових звітів.
* **Автокореляція:** Створено функцію для генерації псевдовипадкових часових рядів із заданим коефіцієнтом автокореляції.

### Блок 5: Інструменти Big Data
* **Читання даних:** Проведено бенчмарк швидкості завантаження CSV. `Polars` працює суттєво швидше за Pandas на багатоядерних системах.
* **Кешування:** Проміжні результати збережено у форматі стовпчикової бази `.parquet` через `PyArrow`.
* **Кластеризація та Ознаки:** Застосовано Dask для обробки даних поза оперативною пам'яттю та `Featuretools` для генерації ознак (Automated Feature Engineering).

##  Як запустити локально

1. Встановіть необхідні залежності:
   `pip install pandas numpy matplotlib numba scipy statsmodels faker markovify dask vaex pyarrow polars featuretools scikit-learn cython`
2. Скомпілюйте Cython-модуль:
   `python setup.py build_ext --inplace`
3. Запустіть основний скрипт:
   `python energy_pipeline.py`
